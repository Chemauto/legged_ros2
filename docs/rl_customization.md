# How to Modify RL-Related Parts in legged_ros2

This guide focuses on modifying the RL pipeline in `legged_ros2`, especially observations and action interfaces.

[TOC]

## 1. RL Runtime Components (Current Code)

Main files involved:

- Controller entry: `legged_rl_controller/src/legged_rl_controller.cpp`
- Env container: `isaaclab/envs/manager_based_rl_env.h`
- Observation registry: `isaaclab/envs/mdp/observations/observations.h`
- Observation manager: `isaaclab/manager/observation_manager.h`
- Action registry: `isaaclab/envs/mdp/actions/joint_actions.h`
- Robot state adapter: `legged_rl_controller/legged_articulation.hpp`
- Policy assets: `<robot_description>/config/rl_policy/policy.onnx` + `IO_descriptors.yaml`

## 2. Before You Change Anything

Keep these invariants true:

1. ONNX input names must match observation group names produced by `ObservationManager`.
2. Observation term names in `IO_descriptors.yaml` must be registered in `observations.h`.
3. Action term names in `IO_descriptors.yaml` must be registered in `joint_actions.h`.
4. Action dimension must match robot joint command dimension.
5. `rl_controller.joint_names` order in `config/ros2_control/*.yaml` must match the policy/export order in `IO_descriptors.yaml` (especially `articulations.robot.joint_names`).

## 3. Common Change Scenarios

## 3.1 Only updating policy weights

If observation/action schema is unchanged:

1. Replace `policy.onnx`.
2. Replace `IO_descriptors.yaml` with the one from the same export.
3. Do not modify C++ code.

## 3.2 Adding or changing observations

This is the most common code change path.

### Step A: export/update descriptors in Isaac Lab

- Update your Isaac Lab observation config and export again.
- Use the exported `IO_descriptors.yaml` directly.

### Step B: implement/register observation term in C++

In `observations.h`, add a new term:

- Use `REGISTER_OBSERVATION(<name>)`.
- `<name>` must exactly match `observations.policy[*].name` in descriptors.
- Return `std::vector<float>` with deterministic size.

### Step C: ensure robot data exists

If the new observation needs data not currently exposed (e.g., extra kinematics), update:

- `legged_articulation.hpp` (`update()` path)
- and/or `ArticulationData` in `isaaclab/assets/articulation/articulation.h`

## 3.3 Changing actions

Current action term is `joint_position_action` in `joint_actions.h`.

If action semantics change:

1. Update/export action descriptors from Isaac Lab.
2. Add or modify action term class in `joint_actions.h`.
3. Ensure class registration name matches descriptor action `name`.
4. Confirm processed action dimension equals controlled joint count.

## 4. Important Constraints in Current Implementation

From `observation_manager.h`:

- Each observation term must have `overloads` in YAML.
- `flatten_history_dim` must be `true`.
- `history_length <= 0` is treated as `1`.
- If `observations` has only one group, it is mapped to input key `"obs"`.

From `legged_rl_controller.cpp`:

- `onnx_model_path` and `io_descriptors_path` cannot be empty.
- `articulations.robot` must exist in descriptors.
- Joint-name count consistency is validated.
- Runtime currently validates joint-name **size**, not exact order; therefore order consistency must be guaranteed by configuration.
- At least one IMU interface is required.
- Action size mismatch causes runtime error.

## 5. IO Descriptors Policy

`IO_descriptors.yaml` is expected to be generated by Isaac Lab export.

Recommended workflow:

1. Train/update policy in Isaac Lab.
2. Export `policy.onnx` and `IO_descriptors.yaml` together.
3. Copy both files into your robot description package.
4. Update launch arguments only if paths changed.

Avoid manual editing unless debugging internal details.

## 6. Test Checklist After RL Changes

1. Build:

```bash
colcon build --packages-select legged_rl_controller go2_description
```

2. Bringup RL stack.
3. Confirm controller config succeeds (no YAML/schema errors).
4. Confirm no runtime errors for:
   - missing observation term registration
   - ONNX input name not found in observation map
   - action size mismatch
5. Verify robot behavior at low command magnitudes first.

## 7. Typical Debug Signals

- `Observation term '<name>' is not registered.`
  - Add matching `REGISTER_OBSERVATION(<name>)`.

- `Input name <...> not found in observations.`
  - ONNX input names and observation group names are inconsistent.

- `Action size mismatch: action=..., joints=...`
  - Action head dimension or joint list does not match runtime robot config.
